🔒 Risk-Free Sandbox Deployment

Deploying a Safe Testing Subpage
to Firebase Hosting

As seen in your Firebase panel (image_e2d88d.jpg), your current live releases are tied directly to your domain handles. We are going to deploy an isolated test subpage (resume-test.html) so you can verify its visual and structural components live on your phone or browser with absolute zero risk of impacting your primary pages.

01

Create the Sandboxed Subpage File

Open your local code editor (VS Code) and right-click inside your subfolder path to create a separate testing sandbox file:

C:\AI_Portfolio\public\forward-moves-ai\resume-test.html

Copy the experimental or newly developed code (like the text-area optimization portal layout we analyzed earlier) and paste it entirely inside this safe new resume-test.html file.

02

Configure Firebase Routing Parameters

Open your master configuration file (firebase.json) in your root folder. Add a safe rewrite rule pointing to our new test page so it is indexed cleanly. 

{
  "source": "/resume-test",
  "destination": "/forward-moves-ai/resume-test.html"
}

Add this block directly inside your "rewrites" array. This guarantees that visiting /resume-test will cleanly serve your test file from behind the scenes!

03

Run the Build & Deploy Commands

Open your VS Code terminal window. Ensure you are sitting at your master project root directory and execute the safe hosting-only deployment pipeline:

firebase deploy --only hosting
💡

Pro-Tip: Using the --only hosting flag is a critical time-saver here. It completely bypasses your Cloud Functions, skipping unnecessary billing configurations, and instantly pushes your static HTML pages to the cloud in under 10 seconds.

04

Verify and Test Live in Production

Once your terminal prints out ✔ Deploy complete!, open a fresh Incognito or Private tab and navigate straight to your staging link:

https://yiynew-e0a98.web.app/forward-moves-ai/resume-test.html
